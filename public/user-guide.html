Overview

The Run Sheet Generator reads scanned oil and gas deed and lease PDFs, automatically extracts legal instruments using AI, and produces a formatted Chain of Title Excel spreadsheet in a fraction of the time it would take manually.

Upload your PDFs, review and correct the extracted data, then export a ready-to-use Excel file sorted by your preferred date field.

Bulk uploads supported. You can process up to 28 or more documents at once. Each file is handled in parallel so processing time does not multiply with file count.

Step 2 — Upload Your PDFs

Click the Upload PDFs file picker and select one or more PDF files. Hold Ctrl (Windows) or Cmd (Mac) to select multiple files at once.

No size limit. Files upload directly to secure cloud storage and never pass through a size-restricted server endpoint.
PDFs only. The tool only accepts .pdf files. Images and Word documents are not supported.

Confidence Color Coding

Each row is color-coded based on how confident the AI is in its extraction. Always review low-confidence rows before exporting.

High

All fields extracted cleanly. Minimal review needed.

Medium

Most fields found but something was unclear. Review recommended.

Low

Significant uncertainty. Always review manually before exporting.

Notes for Reviewer column: When confidence is medium or low, check this column — the AI explains exactly what it was uncertain about.

Tips and Best Practices

For best OCR results

Higher DPI scans produce better results. Scan at 300 DPI or above when possible.
Straighten pages before scanning. Rotated or skewed pages reduce OCR accuracy on handwritten documents.

For large batches

Name files descriptively. The source filename appears in the review table, so names like DB_261-445.pdf make it easier to trace back to the original.
Up to 28 or more files work reliably. Files are processed in parallel so a 20-file batch takes roughly the same time as a 5-file batch.

Reviewing results

Always check low-confidence (pink) rows. Cross-reference against the original PDF before exporting.
A single PDF may produce multiple rows. If a deed book page contains two instruments, both will be extracted as separate rows. Delete any that are not relevant to your chain.

Frequently Asked Questions

I uploaded 10 files but got 17 rows — is that normal?
Yes. A single PDF can contain multiple recorded instruments. The AI extracts each one as a separate row. Review the Source column to see which file each row came from, and delete any rows that are not part of your chain of title.
A file shows as an error — what should I do?
Check the error message in the Errors section. Common causes: the PDF is corrupted or password-protected, the scan is completely blank, or the file contained no readable text. Try re-scanning or opening the PDF to verify it is readable.
Can I edit the data after extracting?
Yes — every field in the review table is editable. Click any input, make your correction, and the change will be reflected in the export automatically.
What is the difference between Doc Date and Recorded Date sorting?
Doc Date is when the instrument was signed or executed. Recorded Date is when it was filed with the county recorder. For chain of title purposes, Recorded Date is typically preferred. Both columns always appear in the export regardless of which sort you choose.
How long does processing take?
Typically 10 to 30 seconds per file, but since files are processed in parallel, a batch of 20 files takes about the same time as a batch of 5.
Is my data stored anywhere?
PDFs are temporarily stored in Vercel Blob storage during processing only. They are not retained long-term. Extracted data exists only in your browser session — nothing is saved to a database.
