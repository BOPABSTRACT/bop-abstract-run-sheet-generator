Overview

The Run Sheet Generator reads scanned oil and gas deed and lease PDFs, automatically extracts legal instruments using AI, and produces a formatted Chain of Title Excel spreadsheet in a fraction of the time it would take manually.

Upload your PDFs, review and correct the extracted data, then export a ready-to-use Excel file sorted by your preferred date field and order.

Bulk uploads supported. You can process up to 28 or more documents at once. Each file is handled in parallel so processing time does not multiply with file count.
Always review before exporting. The AI is a first-pass tool. Legal descriptions, handwritten documents, and complex instruments will require manual correction.

Step 2 — Upload Your PDFs

Click the Upload PDFs file picker and select one or more PDF files. Hold Ctrl (Windows) or Cmd (Mac) to select multiple files at once.

No size limit. Files upload directly to secure cloud storage and never pass through a size-restricted server endpoint.
PDFs only. The tool only accepts .pdf files. Images and Word documents are not supported.
Name files descriptively. The source filename appears in the review table, so names like DB_261-445.pdf make it easier to trace back to the original document.

Step 3 — Generate with Real Data

Click Generate with Real Data. The tool will:

1
Upload all files to secure storage

All PDFs are uploaded in parallel to Vercel Blob cloud storage.

2
Run OCR on each document

Google Document AI reads each PDF and extracts the raw text, including typed and printed content.

3
Extract instruments using AI

Claude AI identifies each primary recorded instrument and pulls all key fields. Prior references cited within a document are not extracted as separate instruments.

4
Display results for review

All extracted instruments appear in the review table sorted newest first by default, color-coded by confidence level.

Step 4 — Review and Edit Extracted Instruments

Every extracted field is editable directly in the review table. The Grantor, Grantee, Description, Comments, and Notes fields are expandable text boxes — drag the bottom edge to resize, or simply type and the box will grow.

✎
Edit any field

Click directly into any cell to correct values. Text boxes for longer fields can be resized by dragging.

✕
Delete a row

Click the Delete button on any row to remove it from the export. Use this to remove prior references that were incorrectly extracted as separate instruments.

↕
Change sort order

Use the Sort table by and Newest First / Oldest First radio buttons to re-order rows. Default is newest first.

Low confidence rows (red) always need review. Check the Notes for Reviewer column for exactly what the AI flagged. Cross-reference against the original PDF before exporting.
Legal descriptions will often need correction. Complex metes and bounds descriptions, acreage calculations, and exception language should always be verified against the source document.
Older handwritten documents require more editing. OCR accuracy on handwriting is limited. Treat handwritten document extractions as a starting point only.

Confidence Color Coding

Each row is color-coded based on how confident the AI is in its extraction. Always review low-confidence rows before exporting.

High

All fields extracted cleanly from a clear, typed document. Minimal review needed.

Medium

Most fields found but something was unclear — fuzzy scan, ambiguous date, or partial name. Review recommended.

Low

Significant uncertainty — handwriting, heavy degradation, or missing fields. Always review manually before exporting.

Notes for Reviewer column: When confidence is medium or low, this column explains exactly what the AI was uncertain about.

Tips and Best Practices

For best OCR results

Higher DPI scans produce better results. Scan at 300 DPI or above when possible.
Straighten pages before scanning. Rotated or skewed pages reduce OCR accuracy, especially on handwritten documents.

What the AI does well

Typed instruments from the 1950s onward extract reliably with high confidence in most cases.
Standard deed language — grantor, grantee, volume/page, and recorded date — is almost always captured correctly on clear scans.

What the AI struggles with

Legal descriptions are frequently incomplete or inaccurate. Always verify against the source document before exporting.
Handwritten documents require significant manual correction. OCR on cursive handwriting has inherent limitations.
Complex exception and reservation language may be paraphrased rather than precisely recited. Verify any Excepting and Reserving language carefully.

Frequently Asked Questions

I uploaded 10 files but got 17 rows — is that normal?
Yes. A single PDF can contain multiple recorded instruments. The AI extracts each primary instrument as a separate row. Review the Source column to see which file each row came from, and delete any rows that are not part of your chain of title. Prior references cited within a document should not appear as separate rows — if they do, delete them.
The instrument type just says "Deed" — how do I fix it?
The AI is instructed to always specify the full deed type (General Warranty Deed, Special Warranty Deed, etc.). If it returns just "Deed", the deed type was likely not stated in the document, or the scan was unclear. Check the source document and correct manually in the review table.
A prior reference was extracted as its own row — what do I do?
Click Delete on that row. Prior references cited within an instrument (e.g. "see DB 526/25") are not separate instruments and should not appear in the chain of title. The AI is instructed to ignore them, but may occasionally include them on complex documents.
The legal description is wrong or incomplete — what should I do?
Edit it directly in the Description text box in the review table before exporting. Legal descriptions are one of the most difficult things for AI to extract accurately, especially on older documents with complex metes and bounds. Always verify against the source PDF.
A file shows as an error — what should I do?
Check the error message in the Errors section. Common causes: the PDF is corrupted or password-protected, the scan is completely blank, or the file contained no readable text. Try re-scanning or opening the PDF to verify it is readable.
What is the difference between Doc Date and Recorded Date sorting?
Doc Date is when the instrument was signed or executed. Recorded Date is when it was filed with the county recorder. Both dates always appear in the export regardless of which sort field you choose. Recorded Date is the standard for most chain of title run sheets.
How long does processing take?
Typically 10 to 30 seconds per file. Since files are processed in parallel, a batch of 20 files takes about the same time as a batch of 5. Very long or heavily degraded PDFs may take longer.
Is my data stored anywhere?
PDFs are temporarily stored in Vercel Blob storage during processing only. They are not retained long-term. Extracted data exists only in your browser session — nothing is saved to a database. Closing or refreshing the page will clear all extracted data.
